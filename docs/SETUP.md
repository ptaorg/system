# セットアップ手順

この手順は、GoogleフォームとGoogleスプレッドシートを使って、PTA自身が入会申込・同意記録・会員名簿を管理するためのものです。

## 1. PTA専用のGoogleアカウントを用意する

個人アカウントではなく、PTA用の共有Googleアカウントを使うことを推奨します。役員交代時に引き継ぎやすく、個人のGoogleドライブに名簿が残ることを避けやすくなります。

## 2. Googleフォームを作る

最低限、次の質問を作ります。質問文は、Apps Script側の `FIELD_MAP` と一致させる必要があります。

- 保護者氏名：記述式、必須
- お子さまの氏名：記述式
- 学年・組：記述式
- メールアドレス：記述式、必須
- 重要事項に同意します：チェックボックス、必須

フォーム説明欄には、`docs/consent-template.md` を各PTA用に編集して貼り付けます。

## 3. フォーム回答先のスプレッドシートを作る

フォーム編集画面で「回答」タブを開き、スプレッドシートのアイコンから回答先を作成します。このスプレッドシートが管理台帳になります。

## 4. Apps Scriptへコードを貼る

スプレッドシートで「拡張機能」→「Apps Script」を開きます。

既定で入っているコードを削除し、`src/Code.gs` の全文を貼り付けて保存します。必要に応じて `src/appsscript.json` の内容もマニフェストへ反映してください。

## 5. CONFIGを編集する

`Code.gs` 冒頭の `CONFIG` を自分のPTAに合わせます。

```javascript
const CONFIG = {
  ORG_NAME: 'サンプルPTA',
  CONTACT_EMAIL: 'pta@example.com',
  CONSENT_VERSION: '2026-06',
  PAYMENT_GUIDE_TEXT: '会費のお支払い方法は、別途配布済みの案内をご確認ください。',
};
```

同意文を改訂したら、`CONSENT_VERSION` も変更してください。以後の申込に新しい版が記録されます。

## 6. FIELD_MAPを確認する

`FIELD_MAP` の右側は、Googleフォームの質問文と一字一句一致させてください。

```javascript
const FIELD_MAP = {
  guardianName: '保護者氏名',
  childName: 'お子さまの氏名',
  gradeClass: '学年・組',
  email: 'メールアドレス',
  consent: '重要事項に同意します',
};
```

ここがずれると、申込者名やメールアドレスが名簿に転記されません。

## 7. 初期設定を実行する

Apps Script画面上部の関数選択で `setupPtaTool` を選び、実行します。初回はGoogleの承認画面が出ます。承認後、スプレッドシートに次のシートが作成されます。

- 申込・名簿
- 会員名簿

## 8. フォーム送信トリガーを設定する

Apps Script左側の「トリガー」から、次のトリガーを追加します。

- 実行する関数：`onFormSubmit`
- イベントのソース：スプレッドシートから
- イベントの種類：フォーム送信時

定期的に支払い案内メールを送りたい場合だけ、次の時間主導型トリガーを追加します。

- 実行する関数：`sendPaymentGuides`
- イベントのソース：時間主導型
- 頻度：各PTAの運用に合わせる

## 9. テスト送信する

Googleフォームからテスト送信します。スプレッドシートの「申込・名簿」に次のように記録されれば成功です。

- 状態：申込受付（入金前）
- 同意：同意
- 同意文の版：`CONFIG.CONSENT_VERSION` の値

メールアドレスを入力している場合は、受付メールも送信されます。

## 10. 入金確認後に会員確定する

会計担当者が現金や振込を確認したら、「申込・名簿」シートで対象行を選択し、メニューの「PTAツール」→「選択した行を会員として確定（入金確認）」を実行します。

状態が「会員（入金確認済）」になり、「入金確認日／会員成立日」が入ります。

## 11. 会員名簿を書き出す

メニューの「PTAツール」→「会員名簿を書き出す（入金確認済のみ）」を実行します。

「会員名簿」シートには、入金確認済みの人だけが出力されます。申込受付中、取下げ、誤送信、重複申込は出力されません。

## 12. 取下げ・誤送信・重複を処理する

対象行を選び、「PTAツール」→「選択した行を取下げ／無効にする」を実行します。

この状態の行は、会員名簿には出ません。

## 注意点

無料のGmailアカウントでは、Apps Scriptから送れるメール数に上限があります。人数が多い場合は、支払い案内メールを複数日に分けるか、Google Workspace等の利用を検討してください。

学校が保有する児童・保護者名簿をPTAへ流用する前提ではありません。PTA自身がフォームで取得した情報だけを使う設計にしてください。

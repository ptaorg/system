# セットアップ手順

この手順は、GoogleフォームとGoogleスプレッドシートを使って、PTA自身が入会申込、重要事項への同意、会費の入金確認、会員名簿を管理するためのものです。

GitHub上でファイルを見ているだけでは動きません。`src/Code.gs` を、フォーム回答先スプレッドシートの Apps Script に貼り付けて使います。

## 1. PTA専用のGoogleアカウントを用意する

個人アカウントではなく、PTA用の共有Googleアカウントを使うことを推奨します。役員交代時に引き継ぎやすく、個人のGoogleドライブに名簿が残ることを避けやすくなります。

## 2. Googleフォームを作る

最低限、次の質問を作ります。質問文は、Apps Script側の `FIELD_MAP` と一致させる必要があります。

```text
保護者氏名
児童・生徒氏名
学年・組
メールアドレス
加入意思の確認
重要事項に同意します
```

推奨する質問形式は次のとおりです。

| 質問 | 形式 | 必須 | 備考 |
|---|---|---:|---|
| 保護者氏名 | 記述式 | 必須 | 申込者本人の氏名 |
| 児童・生徒氏名 | 記述式 | 任意または必須 | PTAの運用に応じて判断 |
| 学年・組 | 記述式 | 任意または必須 | 年度更新時に見直す |
| メールアドレス | 記述式 | 必須 | 受付・連絡用 |
| 加入意思の確認 | ラジオボタン | 必須 | 「PTAへの加入を申し込みます」など |
| 重要事項に同意します | チェックボックス | 必須 | 説明文を読んだ確認 |

`加入意思の確認` の選択肢例は次のとおりです。

```text
PTAへの加入を申し込みます
今回は申し込みません
```

`重要事項に同意します` のチェックボックスは、次のような文言にします。

```text
上記の説明、会費、個人情報の取扱い、退会方法を確認し、同意します
```

フォーム説明欄には、`docs/consent-template.md` を各PTA用に編集して貼り付けます。

## 3. フォーム回答先のスプレッドシートを作る

Googleフォーム編集画面で「回答」タブを開き、スプレッドシートのアイコンから回答先を作成します。このスプレッドシートが申込管理台帳になります。

## 4. Apps Scriptへコードを貼る

回答先スプレッドシートを開き、上部メニューから次の順で開きます。

```text
拡張機能
→ Apps Script
```

既定で入っているコードを削除し、`src/Code.gs` の全文を貼り付けて保存します。

必要に応じて、Apps Scriptのプロジェクト設定から `appsscript.json` を表示し、`src/appsscript.json` の内容を反映してください。通常運用では、まず `Code.gs` の貼り付けだけで足ります。

## 5. CONFIGを編集する

`Code.gs` 冒頭の `CONFIG` を自分のPTAに合わせます。

```javascript
const CONFIG = {
  ORG_NAME: 'サンプルPTA',
  CONTACT_EMAIL: 'pta@example.com',
  CONSENT_VERSION: '2026-06',
  ROSTER_SHEET: '申込・名簿',
  MEMBER_SHEET: '会員名簿',
  EMAIL_ENABLED: false,
  PAYMENT_GUIDE_TEXT: '会費のお支払い方法は、別途配布済みの案内をご確認ください。',
  APPLY_VALUE: 'PTAへの加入を申し込みます',
};
```

同意文や説明文を改訂したら、`CONSENT_VERSION` も変更してください。以後の申込に新しい版が記録されます。

メール送信は初期状態で無効です。テスト完了後に使う場合だけ、次のように変更します。

```javascript
EMAIL_ENABLED: true,
```

## 6. FIELD_MAPを確認する

`FIELD_MAP` の右側は、Googleフォームの質問文と一字一句一致させてください。

```javascript
const FIELD_MAP = {
  guardianName: '保護者氏名',
  childName: '児童・生徒氏名',
  gradeClass: '学年・組',
  email: 'メールアドレス',
  joinIntent: '加入意思の確認',
  consent: '重要事項に同意します',
};
```

ここがずれると、申込者名、メールアドレス、同意記録が名簿に転記されません。

## 7. 初期設定を実行する

Apps Script画面上部の関数選択で `setupPtaTool` を選び、実行します。

初回はGoogleの承認画面が出ます。承認後、スプレッドシートに次のシートが作成されます。

```text
申込・名簿
会員名簿
使い方
```

## 8. フォーム送信トリガーを設定する

Apps Script左側の「トリガー」から、次のトリガーを追加します。

```text
実行する関数: onFormSubmit
イベントのソース: スプレッドシートから
イベントの種類: フォーム送信時
```

この設定をしないと、フォーム送信時に「申込・名簿」へ転記されません。

## 9. テスト送信する

Googleフォームからテスト送信します。

「申込・名簿」に次のような記録が作られれば成功です。

```text
状態: 申込受付（入金前）
同意: 同意
同意文の版: CONFIG.CONSENT_VERSION の値
受付番号: PTA-00001 など
```

加入意思確認で「今回は申し込みません」を選んだ場合は、状態が「申込なし（記録のみ）」になります。

## 10. 入金確認後に会員確定する

会計担当者が現金や振込を確認したら、「申込・名簿」シートで対象行を選択し、メニューから次を実行します。

```text
PTA入会管理
→ 選択した行を会員として確定（入金確認）
```

状態が「会員（入金確認済）」になり、「入金確認日／会員成立日」が入ります。

## 11. 会員名簿を書き出す

メニューから次を実行します。

```text
PTA入会管理
→ 会員名簿を書き出す（入金確認済のみ）
```

「会員名簿」シートには、入金確認済みの人だけが出力されます。申込受付中、申込なし、取下げ、誤送信、重複申込は出力されません。

## 12. 取下げ・誤送信・重複を処理する

対象行を選び、メニューから次を実行します。

```text
PTA入会管理
→ 選択した行を取下げ／無効にする
```

この状態の行は、会員名簿には出ません。

## 13. 支払い案内メールを使う場合

支払い案内メールは、申込受付中の人への案内です。未納者への督促ではありません。

メール送信を使う場合は、テストが終わってから `CONFIG.EMAIL_ENABLED` を `true` にします。その後、メニューから次を実行します。

```text
PTA入会管理
→ 申込受付中の方へ支払い案内メールを送信
```

無料のGmailアカウントでは、Apps Scriptから送れるメール数に上限があります。人数が多い場合は、複数日に分けて送信するなど、各PTAで運用を決めてください。

## 14. 重要な注意点

このツールは、学校が保有する児童・保護者名簿をPTAへ流用する前提ではありません。PTA自身がフォームで取得した情報だけを使う設計にしてください。

取得項目は、入会申込、会費確認、会員連絡に必要な範囲に限定してください。不要な家族情報、勤務先、詳細な生活状況、診断書等の提出を前提にしないでください。

実際の運用では、各PTAの規約、会費額、返金方針、退会方法、会計処理に合わせて文面を調整してください。

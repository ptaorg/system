# 日常運用手順

この手順は、初期設定後にPTA役員や会計担当者が日常的に使うためのものです。

## 1. 申込を確認する

Googleフォームから送信があると、回答先スプレッドシートの通常のフォーム回答シートとは別に、「申込・名簿」シートへ次の情報が記録されます。

```text
受付番号
受付日時
保護者氏名
児童・生徒氏名
学年・組
メールアドレス
加入意思
同意
同意文の版
状態
```

状態が「申込受付（入金前）」であれば、入会申込として受け付けた段階です。この時点では、まだ会員確定ではありません。

## 2. 入金確認をする

会計担当者が現金、振込、その他の方法で会費の受領を確認します。この確認はツールが自動で行うものではありません。

確認後、「申込・名簿」シートで対象行を選び、次を実行します。

```text
PTA入会管理
→ 選択した行を会員として確定（入金確認）
```

状態が「会員（入金確認済）」に変わり、「入金確認日／会員成立日」が入ります。

## 3. 会員名簿を作る

必要な時点で、次を実行します。

```text
PTA入会管理
→ 会員名簿を書き出す（入金確認済のみ）
```

「会員名簿」シートには、入金確認済みの人だけが出力されます。

## 4. 取下げ・誤送信・重複申込を処理する

入会を取りやめた人、誤って送信した人、同じ人の重複申込がある場合は、対象行を選び、次を実行します。

```text
PTA入会管理
→ 選択した行を取下げ／無効にする
```

この状態の行は、会員名簿には出力されません。

## 5. 申込なしとして扱う

加入意思確認欄で「今回は申し込みません」が選ばれた場合は、自動的に「申込なし（記録のみ）」になります。

後から状態を直す場合は、対象行を選び、次を実行します。

```text
PTA入会管理
→ 選択した行を申込なしにする
```

## 6. 支払い案内メールを送る

メール送信を使う場合は、`CONFIG.EMAIL_ENABLED` を `true` にしたうえで、次を実行します。

```text
PTA入会管理
→ 申込受付中の方へ支払い案内メールを送信
```

このメールは、申込受付中の人へ支払い方法を案内するものです。会員確定前の人を当然の債務者として扱う趣旨ではありません。

## 7. 集計を確認する

次を実行すると、状態別の件数を確認できます。

```text
PTA入会管理
→ 状況の集計を表示
```

表示される状態は次のとおりです。

```text
申込受付（入金前）
会員（入金確認済）
申込なし（記録のみ）
取下げ／無効
確認要
その他
```

## 8. 年度更新時の注意

年度が変わると、学年・組、会費額、活動内容、同意文の版を見直す必要があります。

年度更新時は、少なくとも次を確認してください。

```text
フォーム説明文
会費額
徴収方法
退会方法
個人情報の利用目的
CONFIG.CONSENT_VERSION
FIELD_MAP
```

前年度の名簿をそのまま新年度の会員名簿として扱うのではなく、各PTAの規約と運用に従って、継続意思確認や年度更新手続きを整理してください。

PTA入会申込・会員名簿管理ツール

このツールは、GoogleフォームとGoogleスプレッドシートを使って、PTA自身が入会申込、重要事項への同意、会費の入金確認、会員名簿を管理するためのものです。

GitHub上でファイルを見ているだけでは動きません。実行場所はGoogleスプレッドシートです。src/Code.gs を、フォーム回答先スプレッドシートの Apps Script に貼り付けて使います。

1. PTA専用のGoogleアカウントを用意する

個人アカウントではなく、PTA用の共有Googleアカウントを使うことを推奨します。役員交代時に引き継ぎやすく、個人のGoogleドライブに名簿が残ることを避けやすくなります。

2. Googleフォームを作る

最低限、次の質問を作ります。質問文は、Apps Script側の FIELD_MAP と一致させる必要があります。

保護者氏名
児童・生徒氏名
学年・組
メールアドレス
加入意思の確認
重要事項に同意します
質問形式必須備考
保護者氏名記述式必須申込者本人の氏名
児童・生徒氏名記述式任意または必須PTAの運用に応じて判断
学年・組記述式任意または必須年度更新時に見直す
メールアドレス記述式必須受付・連絡用
加入意思の確認ラジオボタン必須「PTAへの加入を申し込みます」など
重要事項に同意しますチェックボックス必須説明文を読んだ確認

3. 回答先スプレッドシートを作る

Googleフォーム編集画面で「回答」タブを開き、スプレッドシートのアイコンから回答先を作成します。このスプレッドシートが申込管理台帳になります。

4. Apps Scriptへコードを貼る

回答先スプレッドシートを開き、拡張機能 → Apps Script を開きます。既定で入っているコードを削除し、src/Code.gs の全文を貼り付けて保存します。

5. CONFIGを編集する

Code.gs 冒頭の CONFIG を自分のPTAに合わせます。

const CONFIG = {
  ORG_NAME: 'サンプルPTA',
  CONTACT_EMAIL: 'pta@example.com',
  CONSENT_VERSION: '2026-06',
  ROSTER_SHEET: '申込・名簿',
  MEMBER_SHEET: '会員名簿',
  EMAIL_ENABLED: false,
  PAYMENT_GUIDE_TEXT: '会費のお支払い方法は、別途配布済みの案内をご確認ください。',
  APPLY_VALUE: 'PTAへの加入を申し込みます',
};

メール送信は初期状態で無効です。テスト完了後に使う場合だけ EMAIL_ENABLED: true に変更してください。

6. FIELD_MAPを確認する

FIELD_MAP の右側は、Googleフォームの質問文と一字一句一致させてください。ここがずれると、申込者名、メールアドレス、同意記録が名簿に転記されません。

const FIELD_MAP = {
  guardianName: '保護者氏名',
  childName: '児童・生徒氏名',
  gradeClass: '学年・組',
  email: 'メールアドレス',
  joinIntent: '加入意思の確認',
  consent: '重要事項に同意します',
};

7. setupPtaToolを実行する

Apps Script画面上部の関数選択で setupPtaTool を選び、実行します。初回はGoogleの承認画面が出ます。承認後、スプレッドシートに「申込・名簿」「会員名簿」「使い方」が作成されます。

8. onFormSubmitトリガーを設定する

Apps Script左側の「トリガー」から、次のトリガーを追加します。

実行する関数: onFormSubmit
イベントのソース: スプレッドシートから
イベントの種類: フォーム送信時

この設定をしないと、フォーム送信時に「申込・名簿」へ転記されません。

9. テスト送信する

Googleフォームからテスト送信します。「申込・名簿」に受付番号、状態、同意文の版が記録されれば成功です。

10. 入金確認後に会員確定する

会計担当者が現金や振込を確認したら、「申込・名簿」シートで対象行を選択し、PTA入会管理 → 選択した行を会員として確定（入金確認） を実行します。

11. 会員名簿を書き出す

PTA入会管理 → 会員名簿を書き出す（入金確認済のみ） を実行すると、「会員名簿」シートには入金確認済みの人だけが出力されます。

12. 取下げ・誤送信・重複を処理する

対象行を選び、PTA入会管理 → 選択した行を取下げ／無効にする を実行します。この状態の行は、会員名簿には出ません。

注意点

このツールは、学校が保有する児童・保護者名簿をPTAへ流用する前提ではありません。PTA自身がフォームで取得した情報だけを使う設計にしてください。

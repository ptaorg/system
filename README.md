# PTA入会申込・会員名簿管理ツール

PTAの入会申込、重要事項への同意、会費の入金確認、会員名簿作成を、Googleフォーム、Googleスプレッドシート、Google Apps Scriptだけで行うための配布用ツールです。

このリポジトリをGitHubで開いているだけでは、ツールは動きません。実行場所はGoogleスプレッドシートです。`src/Code.gs` をスプレッドシートの Apps Script に貼り付け、`setupPtaTool` を実行して使います。

## このツールで解決したいこと

PTAは学校に自動的に紐づく組織ではなく、保護者・教職員が任意に加入する団体です。そのため、会員管理では、少なくとも次の区別を曖昧にしないことが重要です。

第一に、学校が保有する児童・保護者名簿と、PTAが自ら取得する入会申込情報は別物です。学校名簿をそのままPTA会員名簿として使う設計にすると、本人の申込、利用目的、第三者提供、会費徴収の根拠が不明確になります。

第二に、フォームを送信しただけの状態と、会費の入金確認後に会員として確定した状態は別物です。このツールでは、フォーム送信後の初期状態を「申込受付（入金前）」とし、会計担当者が入金を確認した後に、手動で「会員（入金確認済）」へ変更します。

第三に、申込を取り下げた人、誤送信、重複申込、加入申込ではない回答は、会員名簿に出さない必要があります。このツールでは、これらを「取下げ／無効」または「申込なし（記録のみ）」として扱い、会員名簿の書き出し対象から除外します。

## できること

- Googleフォームからの入会申込受付
- 重要事項説明への同意記録
- 受付番号の自動採番
- 申込受付、申込なし、確認要、取下げ、会員確定の状態管理
- 入金確認後の会員確定
- 入金確認済みの人だけを会員名簿へ書き出し
- 申込受付中の人への支払い案内メール送信
- 運用状況の集計表示

## できないこと

- 決済代行
- 銀行振込や現金の自動照合
- 学校名簿の自動取り込み
- 規約や会計処理の自動作成
- 法的助言の代替

会費の受領確認は、現金、振込、その他の方法に応じて、各PTAの会計担当者が別途行ってください。このツールは、お金の移動そのものを扱わず、入金確認の記録だけを残します。

## 最短手順

1. PTA専用のGoogleアカウントを用意する。
2. Googleフォームを作る。
3. フォームの回答先スプレッドシートを作る。
4. スプレッドシートで「拡張機能」→「Apps Script」を開く。
5. `src/Code.gs` の全文を `Code.gs` に貼り付ける。
6. `CONFIG` と `FIELD_MAP` を自分のPTAに合わせる。
7. `setupPtaTool` を1回実行する。
8. `onFormSubmit` を「スプレッドシートから」「フォーム送信時」のトリガーに設定する。
9. Googleフォームからテスト送信する。
10. スプレッドシートの「申込・名簿」に記録されることを確認する。

詳しくは `docs/SETUP.md` を参照してください。

## 初期状態でメール送信を止めている理由

`CONFIG.EMAIL_ENABLED` は初期値で `false` です。これは、設定前のテスト送信や誤操作で、申込者へメールが送られることを防ぐためです。

フォーム項目、団体名、問い合わせ先、支払い案内文を確認し、テスト送信で記録が正しく作られることを確認した後、メール送信を使う場合だけ `true` に変更してください。

```javascript
EMAIL_ENABLED: true,
```

## 基本設計

このツールは、PTA会員管理を次の流れで扱います。

```text
重要事項説明
→ 本人による申込
→ 申込受付（入金前）
→ 会費の入金確認
→ 会員（入金確認済）
→ 会員名簿へ書き出し
```

フォーム送信だけで会員にしないこと、学校名簿を当然にPTA名簿へ転用しないこと、会費の確認前に会員確定として扱わないことが基本です。

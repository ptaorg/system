# PTA運営支援ツール

Googleフォーム、Googleスプレッドシート、Apps Scriptだけで動く、PTA入会申込・重要事項同意・入金確認・会員名簿作成のための最小構成です。

このツールは、PTAを「学校に自動的に紐づく組織」ではなく、保護者・教職員が任意で加入する団体として扱うための実務用のたたき台です。学校が保有する児童・保護者名簿を前提にせず、PTA自身がGoogleフォームで取得した申込情報だけを使います。

## このツールの考え方

入会は、重要事項の説明、本人による申込、会費支払いの確認を経て記録されます。フォーム送信だけで直ちに会員にするのではなく、初期状態は「申込受付（入金前）」です。PTA側が入金を確認した時点で「会員（入金確認済）」に変更し、その日を会員成立日として残します。

この設計により、「全員加入」「みなし加入」「学校名簿からの流用」「会費徴収だけ先行」といった運用を避け、申込・同意・会員確定の証跡を一件ずつ残せます。

## できること

- Googleフォームによる入会申込
- 重要事項説明への同意記録
- 申込受付番号の自動採番
- 入金確認後の会員確定
- 入金確認済みの人だけの会員名簿作成
- 申込受付中の人への支払い案内メール
- 取下げ・誤送信・重複申込の無効化
- 任意でLINE公式アカウントによる一斉連絡

## できないこと

- 決済代行
- 銀行口座や現金の自動照合
- 学校名簿からの自動取り込み
- LINEでの個別配信
- 法的助言の代替

会費の実際の受領は、現金、振込、その他の方法を各PTAで決めてください。このツールはお金の移動を扱わず、入金確認の状態を記録するだけです。

## 使い始める手順

詳しい手順は `docs/SETUP.md` を見てください。基本の流れは次のとおりです。

1. PTA専用のGoogleアカウントを用意する
2. Googleフォームを作る
3. フォーム回答先のスプレッドシートを作る
4. スプレッドシートのApps Scriptに `src/Code.gs` を貼る
5. `CONFIG` と `FIELD_MAP` を自分のPTAに合わせる
6. `setupPtaTool` を1回実行する
7. `onFormSubmit` のフォーム送信トリガーを設定する
8. テスト送信し、「申込・名簿」シートに記録されることを確認する

## 運用上の注意

学校が持つ児童・保護者名簿をPTA名簿として使う前提にはしていません。申込フォームからPTA自身が取得した情報だけで管理してください。

「支払い案内メール」は、申込受付中の人への案内です。会員確定前の人を当然の債務者として扱う趣旨ではありません。入会を取りやめる場合や誤送信の場合は、取下げ／無効にして会員名簿へ出さない運用にしてください。

重要事項説明や会費返還の扱いは、各PTAの規約・会計処理・実際の活動内容に合わせて調整してください。
